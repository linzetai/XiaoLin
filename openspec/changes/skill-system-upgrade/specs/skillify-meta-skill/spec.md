## ADDED Requirements

### Requirement: Skillify converts conversation patterns to skills
The system SHALL provide a `/skillify` command or agent tool that extracts reusable patterns from conversation history and creates a new SKILL.md.

#### Scenario: Skillify from conversation
- **WHEN** the user invokes `/skillify` or the agent decides to create a skill from a successful pattern
- **THEN** the system SHALL analyze the recent conversation turns
- **AND** extract the key instructions, tool usage patterns, and constraints
- **AND** generate a SKILL.md with appropriate frontmatter (name, description, tags)

#### Scenario: Skillify generates valid frontmatter
- **WHEN** a skill is generated by skillify
- **THEN** the SKILL.md SHALL have YAML frontmatter with at minimum: name, description
- **AND** if specific tools were used, the `tools` field SHALL list them
- **AND** if the pattern is project-type-specific, `paths` SHALL include relevant globs

### Requirement: Skillify saves to project skills
The generated skill SHALL be saved to the project-level skill directory.

#### Scenario: Save to project directory
- **WHEN** skillify generates a skill
- **THEN** it SHALL be saved to `<workspace>/.xiaolin/skills/<generated-id>/SKILL.md`
- **AND** the skill registry SHALL be hot-reloaded

#### Scenario: User review before save
- **WHEN** skillify generates a skill
- **THEN** the full content SHALL be displayed to the user for review
- **AND** the user SHALL confirm or edit before the skill is saved

### Requirement: Skillify prompt template
The system SHALL use a dedicated prompt template for skill generation.

#### Scenario: Template produces consistent output
- **WHEN** the skillify prompt processes a conversation excerpt
- **THEN** the generated SKILL.md SHALL follow the standard format (frontmatter + markdown body)
- **AND** the body SHALL include clear instructions that an agent can follow
