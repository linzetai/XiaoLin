## ADDED Requirements

### Requirement: 提供批量文件读取入口

系统 SHALL 提供单次调用读取多个文件的能力（新增 `read_files` 工具，或扩展 `read_file` 接受路径数组），使 agent 在需要理解跨多个文件的上下文时，把 N 次读取往返压缩为 1 次工具调用。

#### Scenario: 单次调用读取多个文件
- **WHEN** agent 传入一组文件路径（如 3-5 个）
- **THEN** 工具 MUST 在一次返回中给出全部文件的内容，每个文件清晰标注其路径边界，便于模型区分

#### Scenario: 部分文件失败不影响其余
- **WHEN** 批量读取中某个路径不存在或不可读
- **THEN** 返回 MUST 对该文件给出结构化错误与恢复提示（遵循错误恢复约定），同时正常返回其余可读文件的内容，而非整体失败

#### Scenario: 复用既有单文件能力
- **WHEN** 批量读取处理每个文件
- **THEN** 每个文件 MUST 复用 `read_file` 既有的工作区校验、大小上限、行号前缀与去重缓存能力，行为与单文件读取一致

### Requirement: 批量读取受聚合预算约束

批量读取的总返回 SHALL 受单消息工具结果预算约束；当多文件内容总量超限时，MUST 按既有大输出策略落盘并返回预览+路径，而非静默截断丢失内容。

#### Scenario: 多文件总量超预算时落盘
- **WHEN** 一次批量读取的合计内容超过结果大小预算
- **THEN** 系统 MUST 走既有落盘+预览机制（提供可后续 `read_file` 访问的路径），保证内容不丢失
